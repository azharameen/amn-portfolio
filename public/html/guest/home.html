LiveFeeds Installation

Include script file in your page.

<script src="http://amn-portfolio.herokuapp.com/js/livefeed.js"></script>

Initiate the LiveFeed class with token.

var feed = new LiveFeed(secret_key);

register to get servet_key so to use our service.

LiveFeed send data to all who are connected to your project. receive the data as callback. it won't save, only send and receive at the current time.

feed.livefeed(callback, data);

Here data is the message that you want to pass to other users connected. set your callback parametr to listen

feed.listen.on(callback, data);

data to be received and callback that you have set

SendFeed send data to all who are connected to your project. receive the data as callback. it will be saved. can send and receive data at any time, data can be stored and used later. Note: Same key request will override the data.

feed.send_feed(callback, key, data);

Here the term key defines any particular user or category in your project, and data defines your message to be saved and passed.

feed.listen.on(callback, data);

callback is a method to receive your temp feeds. data is your temp value

SendMsg send data to a particular user who is connected to your project. receive the data as callback. it won't be saved. can send and receive data only current time, data can't be stored and nor used later. Note: User id should be passed so to send only to particular user.

feed.send_msg(callback, key, data);

Here the term key defines any particular user or browser in your project, and data defines your message to be saved and passed.

feed.listen.on(callback, data);

callback is a method to receive your temp feeds. data is your temp value

Insert method use to store data for future use. Use only object to store data. you can set your own call back function. Note: Data should be passed only as objects, key and value.

feed.insert(callback, table, data);

data - the data you want to insert, table - specify the table you want to use. set your callback parametr to listen.

feed.listen.on(callback, data);

callback is a method to check your data is stored. data is your insert status.

Update method use to update or override data which already stored. Use only object to store data. you can set your own call back function and give a condition as last parameter. Note: Data should be passed only as objects, key and value.

feed.update(callback, table, data, where);

data - the data you want to update, table - specify the table you want to use, where - the conditions you want to set. set your callback parametr to listen.

feed.listen.on(callback, data);

callback is a method to check your data is updated. data is your update status

Fetch method used to retrive stored data. set callback function to get the stored data. Note it will send all the data stored.

feed.send_msg(callback, select, table, where);

select - the fields you want to fetch, table - specify the table you want to use, where - the conditions you want to set. set your callback parametr to listen.

feed.listen.on(callback, data);

callback is a method to receive your stored data. data is your values
